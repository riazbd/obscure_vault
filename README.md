# OBSCURA VAULT
### Automated YouTube engine for buried history
*History They Buried. We Dig It Up.*

A fully local, fully free, autonomous YouTube production pipeline.
Idea → research → script → SEO → narration → smart B-roll → captions →
AI thumbnail → audio polish → branded intro/outro → upload → analytics
feedback. Runs on a Lenovo T480-class laptop with no GPU, no paid APIs,
no recurring bills.

## 1. What it does

Given a topic — or no input at all when run on the scheduler — the
system produces a YouTube-ready 1080p video with everything attached:

- **Narration script** generated by an LLM, optionally grounded in
  Wikipedia + web sources
- **SEO-optimised title** (12 candidates, scored, top picked) +
  description with timestamps + tags + chapters
- **Voiceover** in your choice of authoritative documentary voices
  (free Microsoft Edge TTS, can rotate among multiple to fight monotony)
- **Burned-in captions** for the entire ~50 % of YouTube traffic that
  watches without sound (transcribed locally with `faster-whisper`)
- **Smart B-roll** — script split into ~10 s chunks, each chunk gets
  its own LLM-written cinematographic search query, footage sourced
  from Pexels (and optionally Pixabay), picked by a scoring rubric,
  colour-graded, with a slow Ken-Burns pan across each clip
- **AI thumbnail** — LLM punchline + LLM image prompt → AI image gen
  via Pollinations.ai → PIL composition with channel branding;
  produces 1–3 variants for YouTube's native A/B test
- **Audio polish** — voice normalised to broadcast-spec −16 LUFS,
  background music dynamically ducks 6–10 dB whenever the narrator
  speaks (sidechain compression)
- **Branded intro/outro** — your own MP4 stings auto-pre-pended and
  appended, lossless concat once normalised on upload
- **Auto-upload** to YouTube as private (or public, scheduled, etc.)
  with thumbnail, captions, AI-disclosure flag set per policy
- **Analytics feedback** — daily pull of CTR/AVD/views; per-token
  performance signals reweight the next batch of harvested ideas

There's also a **Shorts pipeline** for vertical 1080×1920 hook-only
content, and a **performance review** that LLM-scores any finished
video against the channel baseline and tells you what to do better.

## 2. Hardware & system requirements

### Minimum (the target machine: ThinkPad T480)
- CPU: i5 7th gen or equivalent (4 cores / 8 threads)
- RAM: 8 GB
- Disk: 256 GB SSD with at least **40 GB free**
- GPU: not required
- Network: anything — needs ~100 MB / video for footage download

### Software
- **Python 3.10+** (3.11 or 3.12 recommended)
- **FFmpeg 4.4+** with libx264 + libfdk-aac (or AAC) + zoompan +
  sidechaincompress + subtitles filters (all standard)
- A modern browser — the UI runs at `http://localhost:5050`

### Disk budget
- Per finished render (kept): ~700 MB–1.5 GB MP4 + ~80 KB thumb + ~20 KB SRT
- Per render (intermediate, auto-cleaned): ~3–5 GB during render
- Cache (research / images / LLM): a few hundred MB over time
- Hard cap: configurable, default 30 GB on the `output/` directory

## 4. Installation

### Step 4.1 — Install Python
- **Windows / Mac:** download from `https://python.org/downloads`. On
  Windows, tick "Add Python to PATH" during install.
- **Linux / WSL:** `sudo apt update && sudo apt install python3 python3-pip -y`

### Step 4.2 — Install FFmpeg
- **Ubuntu / Debian / WSL:** `sudo apt install ffmpeg -y`
- **Mac (Homebrew):** `brew install ffmpeg`
- **Windows:**
  1. Download from `https://ffmpeg.org/download.html` → "Windows builds"
  2. Extract the zip
  3. Copy the `bin/` folder path (e.g. `C:\ffmpeg\bin`)
  4. Start → search "Environment Variables" → Edit System PATH → New →
     paste the path → OK
  5. Restart your terminal and verify: `ffmpeg -version`

### Step 4.3 — Clone & launch
```bash
git clone https://github.com/riazbd/obscure_vault.git
cd obscure_vault
python start.py
```

`start.py` automatically:
- Verifies Python and FFmpeg are present
- Installs the four required Python packages
  (`pillow`, `edge-tts`, `requests`, `flask`)
- Starts the Flask server on port 5050
- Opens your browser

If a `python: command not found` error appears on Windows, reinstall
Python with "Add to PATH" checked.

## 5. First run — 7 steps to your first video

Once `python start.py` opens `http://localhost:5050`:

### 5.1 — Settings → OpenRouter API key
- Sign up at `https://openrouter.ai/keys`. It's free; no card needed.
- Copy the key (starts with `sk-or-v1-…`)
- Paste it into the **OpenRouter API Key** field in Settings →
  click **Validate & Save**. The check sends a 1-token "ok" prompt
  to a free model. A green badge confirms.

### 5.2 — Settings → Pexels API key
- `https://pexels.com/api` → Get Started (free, 1 minute signup)
- Copy your key from the dashboard
- Paste, click **Validate & Save**

### 5.3 — Pick a narrator voice
- **Default voice** dropdown: Guy is recommended for dark history.
- (Optional) tick 2+ voices in the **Rotation pool** to fight
  monotony across many videos.

### 5.4 — Drop some music in the library
- Music tab → drag in 3–10 dark-ambient MP3s. Free sources:
  - `pixabay.com/music` — search "dark ambient", "documentary"
  - `freemusicarchive.org` — filter by CC0
  - `incompetech.com` — Kevin MacLeod, CC-BY (credit required in description)

### 5.5 — (Optional) Install caption deps
- Settings → Burn-in Captions → click **Install caption deps**
- This runs `pip install faster-whisper` (~200 MB) in a background
  job; you'll watch the live log
- Once done, tick **Generate & burn-in captions on every video**.
  Recommended — captions add measurable retention.

### 5.6 — Generate your first video
Two options:

**A. Manual (you provide title + script)**
- Generate Video tab → fill **Video Title** and **Narration Script**
- Click **Generate Video**
- Watch the progress panel; ~25–40 min on the T480

**B. AI-from-topic (recommended)**
- Generate Video tab → top "⚡ Generate Script with AI" card
- Type a topic in **Topic / Idea** (e.g.
  `The 1959 Dyatlov Pass incident and what Soviet investigators hid`)
- Set target length (10 min default)
- Tick **Research first** if you want claims grounded in Wikipedia
- Click **Generate Title + Script + SEO** → watch the log → title and
  script auto-fill below
- Now click **Generate Video** under the Input card

### 5.7 — Review and upload
When done, you'll see the **Result** card with thumbnail preview,
description, and tags. The MP4 + thumbnail + SRT are in `output/`.

To upload:
- (One-time) Settings → YouTube Upload → follow the 3-step setup
  (install libs → upload `client_secrets.json` from Google Cloud →
  Authorize)
- Then on any Library card click **▶ YT** to upload as private
- Or set Settings → "Auto-upload every generated video" on for fully
  unattended operation

## 6. Daily use

### Manual workflow
1. Generate Video tab → topic → script → render → upload (or manual
   YT upload from Library tab).

### Semi-automated
1. Settings → Scheduler → enable **🌾 Harvest ideas** (every 6 h).
   The system trawls YouTube Suggest, Reddit, and Wikipedia.
2. Ideas tab → review the daily harvest, click **▶ Generate** on
   anything that looks good.

### Fully automated (once you trust it)
1. Enable scheduler tasks **Harvest ideas**, **Produce top idea**,
   **Refresh analytics**, and **Storage cleanup**.
2. Turn on **Auto-upload every generated video**.
3. Set a default privacy of **Private** at first so you can spot-check
   each upload before flipping it public.
4. Leave the laptop running. Walk away.

## 7. Settings reference

### OpenRouter API Key
The brain of every LLM-driven feature. Required. Free.

### Pexels API Key
Footage source. Required for any rendered video; without it you get
flat dark cards.

### Pixabay API Key (optional)
A second free footage source. Used by the smart B-roll engine to
diversify per-chunk results.

### Narrator Voice
- **Default voice** — used when the rotation pool is empty
- **Rotation pool** — tick 2+ voices for random rotation per render

### Music Volume + Audio Polish
- **Music volume** — base level (0–50 %)
- **Audio polish** — when on (default), voice is normalised to −16 LUFS
  / −1.5 dBTP and music ducks under voice via sidechain compression
- **Music duck depth** — how many dB the music dips when voice is loud
  (3–18, default 8)

### Burn-in Captions
- **Toggle** — burn captions into the video and write a separate `.srt`
  for YouTube's caption track
- **Whisper model** — `tiny.en` (fastest) / `base.en` (recommended) /
  `small.en` (best, slowest)
- **Install caption deps** — one-click `pip install faster-whisper`
- The whisper model itself (~140 MB for `base.en`) downloads on first
  transcription

### YouTube Upload
- **Install libs** — installs google-api-python-client + google-auth-oauthlib
- **Upload client_secrets.json** — file you downloaded from Google
  Cloud Console (create OAuth Desktop app credentials, enable YouTube
  Data API v3)
- **Authorize** — opens a browser to consent
- **Auto-upload every generated video** — fully unattended publish
- **Default privacy** — private / unlisted / public
- **Disclose AI / synthetic media** — sets `containsSyntheticMedia`
  flag (recommended on, required by YouTube AI policy)

### Auto-translate languages
*(Removed — channel is English-only.)*

### Channel Branding (intro / outro)
- 4 slots: long_intro, long_outro, short_intro, short_outro
- Drop in any MP4/MOV/MKV/WebM; auto-normalised to canvas spec on
  upload (1920×1080 for long, 1080×1920 for shorts, 30 fps, h264, AAC)
- **Apply branding** master toggle

### Smart B-roll
- **Toggle** — when on (default), uses semantic per-chunk LLM-driven
  search instead of legacy substring keyword matching
- **Target chunk length** — seconds per visual cut (default 10)
- **Auto-research every idea before scripting** — toggles the research
  engine for the full Idea-→-Render chain
- **Motion effect** — `pan` (subtle Ken Burns drift, default) /
  `zoom` (centred zoom-in, best for stills) / `off`

### AI Thumbnail
- **Toggle** — use Pollinations + LLM punchline composition
- **Variants per video** — 1–3, saved as `*_v2.jpg`, `*_v3.jpg`

### Video Output
- **Resolution** — 1920×1080 (default) or 1280×720 (faster render)
- **Max footage clips** — used by the legacy keyword-search path

### Storage
- **Auto cleanup workspace** — drop intermediate files after each
  render (default on)
- **Output cap GB** — hard ceiling on total `output/` directory size
  (default 30); when exceeded, oldest MP4s are deleted on next render

### Scheduler
4 tasks, each independently configurable:

| Task | Default interval | What it does |
|---|---|---|
| 🌾 Harvest ideas | 6 h | Mines fresh topics from YT Suggest, Reddit, Wikipedia |
| 🎬 Produce top idea | 12 h | Picks highest ranked_score pending idea, runs full pipeline |
| 📊 Refresh analytics | 24 h | Pulls CTR/AVD/views from YouTube |
| 🧹 Storage cleanup | 24 h | Prunes old workspaces + enforces output cap |

All default off. Click **Run now** to trigger any task immediately.

## 8. Scheduler — full autopilot

A daemon thread starts on boot and ticks every 60 seconds. Each task
runs only when its interval has elapsed since its last successful run.

The **Produce top idea** task is the autopilot heart:
1. Looks at all `pending` ideas in `data/ideas.json`
2. Sorts by `ranked_score = niche_fit × performance_multiplier`
3. Picks the top one
4. Skips if any other render is currently running (no double-rendering
   on the T480)
5. Runs the full pipeline: research → script → SEO → voice → footage →
   captions → thumbnail → render → audio polish → branding → upload
6. Marks the idea as `produced` with the resulting `video_id`

Combined with **Harvest ideas** running every 6 h and **Refresh
analytics** running daily, the system runs itself: it finds new ideas,
ranks them, learns from past performance, and ships the best one
twice a day.

State persists in `data/scheduler.json` so restarts don't double-fire
or skip work.

## 9. Performance expectations

### One 10-minute long-form video on the T480

| Stage | Wall-time |
|---|---|
| Idea research (if enabled) | 1–3 min |
| Script + outline (LLM) | 1–2 min |
| SEO (titles, description, tags) | 30 s |
| Voiceover (Edge TTS) | 30 s |
| Captions (`faster-whisper base.en`, int8) | ~3 min |
| Footage fetch (smart B-roll, 60 chunks) | 2–6 min |
| Footage process + Ken Burns (libx264 fast) | 6–10 min |
| AI thumbnail (Pollinations + composition + QA) | 1–2 min |
| Final assembly (libx264 medium, captions, audio polish) | 8–12 min |
| Storage cleanup | < 5 s |
| Upload (your bandwidth) | 2–4 min |
| **Total** | **~25–40 min** |

3 long-form videos per day per channel without breaking sweat.

### One 30-second Short
~5–10 min total (smaller render, fewer chunks).

### LLM costs
$0. All script + SEO + research + thumbnail-prompting + idea-scoring
calls go to OpenRouter free models. Caching by sha256 of prompt+model
reduces repeats to instant.

## 10. Troubleshooting

### `python: command not found` (Windows)
Reinstall Python from python.org with **"Add Python to PATH"** ticked.

### `ffmpeg: command not found`
Re-read §4.2. The `bin/` folder must be on your system PATH.

### "Cannot connect to server" in the UI
The Flask server isn't running. Run `python start.py` in a terminal
and keep it open.

### Pexels validation says "Invalid"
You either copied a partial key or signed up but didn't activate.
Visit `pexels.com/api`, log in, copy the full key on the dashboard.

### OpenRouter says "all models failed"
Your free-tier daily quota is exhausted, **or** all 5 cascaded models
are temporarily down (rare but happens). Wait ~1 hour and retry.

### Render runs but the video is just a dark screen
Pexels footage download failed (network or rate-limit). Check the
job's log in the Jobs tab — should mention failed downloads. The
pipeline falls back to a dark card so the video still ships.

### "FFmpeg assembly failed: subtitles filter not found"
Your FFmpeg build is too old. Use FFmpeg 4.4+. On Windows, prefer
gyan.dev's "full" build.

### Captions toggle is on but no captions appear
`faster-whisper` isn't installed. Settings → Burn-in Captions →
click **Install caption deps**. First transcription downloads the
model (~140 MB).

### `pollinations failed: HTTP 429`
Pollinations is throttling. The thumbnail engine retries 3× with
backoff; if it still fails, the pipeline falls back to the legacy
frame-based thumbnail.

### YouTube upload fails "quota exceeded"
You've hit YouTube Data API's 10 000 units/day free quota. Each
upload costs ~1 600 units. Wait until UTC midnight or create a
second Google Cloud project for fresh quota.

### `data/jobs.db is locked`
Two pipelines tried to run concurrently. The orphan-recovery on
boot should fix this on next restart; otherwise delete `data/jobs.db`
and the schema will be re-created (you lose history).

### Output directory keeps growing past 30 GB
Either `auto_cleanup_workspace` is off, or `output_cap_gb` is set
high. Settings → Audio / Smart B-roll cards have these toggles.

### Scheduler enabled tasks aren't firing
Check `data/scheduler.json`. The "next_run" field tells you when
the next tick should hit. If a task last erred, status is shown
in the Settings → Scheduler card; "Run now" forces immediate run.

## 11. Architecture

### High level
- **`server.py`** — Flask app, 50+ JSON endpoints, two pipeline
  threads (`run_pipeline_thread` for long-form,
  `run_short_pipeline_thread` for Shorts), idea-to-video chain,
  YouTube upload jobs, scheduler bootstrap
- **`engines/`** — 14 stateless or thread-safe modules, each
  one feature
- **`ui/index.html`** — single 3 000-line file, 9 tabs, all UI state
  in JavaScript, no build step
- **`data/`** — JSON state files + `jobs.db` SQLite + binary caches

### Engines

Each engine has detailed documentation in `docs/<engine>.md`. Quick
overview:

| Engine | Role |
|---|---|
| `llm.py` | OpenRouter chat client with cascade + cache |
| `script.py` | Outline + draft pass for narration |
| `seo.py` | Title scoring + description + tags + chapters |
| `research.py` | Wikipedia + DDG fact extraction |
| `footage.py` | Semantic per-chunk B-roll |
| `captions.py` | Whisper transcription + ASS burn-in |
| `thumbnail.py` | LLM punchline + Pollinations + PIL composition |
| `ideas.py` | Harvest + dedup + score + rank |
| `analytics.py` | YouTube metrics + token signals |
| `upload.py` | YouTube Data API + OAuth |
| `branding.py` | Intro/outro stings |
| `jobs.py` | SQLite job persistence |
| `scheduler.py` | Cron task runner |
| `review.py` | LLM scorecard |
| `storage.py` | Cleanup + cap enforcement |

### Pipeline flow (long-form)
```
idea/topic
  → (optional) research engine → research_pack
  → script engine (outline → draft) → script.txt
  → SEO engine → title + description + tags + chapters
  → voiceover (Edge TTS) → voiceover.mp3
  → captions engine (whisper) → captions.ass + captions.srt
  → footage engine (chunk → query → search → download → grade) → footage_track.mp4
  → music selection (random from /music)
  → final FFmpeg assembly (subtitle burn-in + audio polish + ducking)
  → AI thumbnail (Pollinations + PIL) → thumbnail.jpg
  → metadata.json + description.txt
  → branding concat (intro + main + outro)
  → workspace cleanup
  → output cap enforcement
  → (optional) auto-upload to YouTube
  → analytics records the upload
```

### State files
- `config.json` — every setting in §7
- `data/ideas.json` — harvested topics
- `data/uploads.json` — append-only register of every upload
- `data/metrics.json` — last YouTube analytics pull
- `data/scheduler.json` — per-task last_run / status
- `data/jobs.db` — SQLite, every long-form + Shorts render job
- `data/research_packs/<slug>_<ts>.json` — citation-grounded fact lists
- `data/branding/<slot>.mp4` — normalised intro/outro stings
- `data/youtube/{client_secrets,token}.json` — OAuth credentials

### Caches
- `data/cache/llm/<sha>.json` — LLM responses, keyed by prompt+model+temp
- `data/cache/images/<sha>.png` — Pollinations backgrounds
- `data/cache/research/<sha>.txt` — fetched-and-stripped HTML

## 12. Storage management

The T480 has limited SSD. The pipeline produces ~3–5 GB per render
in intermediates plus a 700 MB–1.5 GB final MP4. Without management,
the disk fills in ~50 renders.

What runs automatically:

1. **Per-render cleanup** — every successful render runs
   `cleanup_workspace(job_name)` which deletes the `processed/` and
   `footage/` sub-directories of that job's workspace, keeping only
   the small text artifacts (`script.txt`, `metadata.json`,
   `outline.json`, `description.txt`, `captions.srt`). Saves ~3 GB
   per render. Toggle: Settings → "Auto cleanup workspace".

2. **Output cap enforcement** — every successful render runs
   `enforce_output_cap(gb)` which deletes the oldest MP4s + their
   thumbnails + SRTs + workspaces until `output/` is under cap.
   Default 30 GB.

3. **Scheduler cleanup** — the optional `storage_cleanup` task runs
   `cleanup_all_workspaces(days=7)` to nuke anything older, plus
   re-enforces the output cap. Default off; enable in Settings →
   Scheduler card.

The Dashboard tab has a Storage card showing real-time disk usage
and a one-click "🧹 Run cleanup now" button.

## 13. Privacy & licensing

- **All keys stored locally** in `config.json`. Never transmitted
  except to the actual upstream service (OpenRouter, Pexels, etc.)
- **No telemetry, no analytics** sent anywhere by this software
- **Cached LLM responses, image responses, research text** are
  on-disk only at `data/cache/`

### Licensing of generated content
- **Edge TTS voices** — free for commercial use per Microsoft TOS
- **Pexels footage** — free for commercial use per Pexels licence
  (no attribution legally required, but consider crediting the channel)
- **Wikipedia text** — CC-BY-SA (your script paraphrases facts, not
  text — research engine extracts atomic claims)
- **Pollinations images** — free, public-domain-ish; check current TOS
  if your channel is commercial

### YouTube AI disclosure
The pipeline sets `containsSyntheticMedia=true` on every upload by
default. You can toggle this off in Settings, but per YouTube policy
AI-generated narration **must** be disclosed for sensitive topics
(news, public figures, current events).
